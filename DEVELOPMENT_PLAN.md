# 开发计划

## 已完成

### 1. 普通用户认证主流程

- 用户名密码登录
- 手机验证码登录
- 手机验证码注册
- 登录态持久化

### 2. 当前用户鉴权收口

- 前端统一通过令牌访问受保护接口
- 后端不再信任前端自行传入 `author`、`admin`、`reporter` 等身份字段

### 3. 管理员体系拆分

- 新增独立管理员表 `admin_user`
- 新增独立管理员会话表 `admin_session`
- 新增独立管理员登录接口
- 新增独立后台入口 `admin.html`
- 主站移除后台入口
- 普通用户不再支持升级为管理员

### 4. 管理台列表分页与工作台改造

- 管理账号、用户、举报、帖子、动态、分类、地区接口支持分页
- 管理台按标签页分区加载，避免所有模块一次性堆叠
- 主站核心列表增加分页展示，提升可读性与浏览效率

### 5. 地区库标准化

- 管理台地区库改为省 / 市 / 区县三级浏览
- 地区库改为只读筛选，不再允许后台手工新增
- 全国地区数据迁移为民政部公开标准行政区划数据

### 6. 管理员权限体系

- 新增超级管理员 / 普通管理员角色边界
- 普通管理员改为按权限码授予后台能力
- 管理台菜单、页面、按钮按权限动态展示
- 后端接口按权限码进行服务端拦截

### 5. 关键验证

- 管理员账号不能通过普通用户登录入口登录
- 管理员可通过独立后台登录
- 前台与后台访问边界已分离
- 分页接口已通过后端测试、前端构建与 E2E 校验

## 进行中

### 1. 历史清理

- 清理旧测试账号与历史脏数据
- 删除主站残留旧后台代码
- 修复历史中文乱码文档与注释

### 2. 后端规范化迁移

- 补齐更多 Request / Response DTO
- 持续收紧 Controller / Service / DTO 边界
- 统一接口返回格式、响应码与错误码
- 完善分页、校验、异常处理等公共能力

### 3. 页面体验优化

- 继续优化主站列表密度、留白与信息层次
- 继续拆分管理台模块布局
- 补齐更清晰的空状态、加载状态、分页操作反馈

## 后续建议优先级

### 高优先级

- 将更多列表接口迁移为数据库层分页，而不是内存切片
- 管理台补管理员停用、重置密码、修改显示名
- 清理 `frontend/src/main.tsx` 中残留的旧后台历史代码
- 将地区库主站使用链路逐步统一到 `areaCode` 维度

### 中优先级

- 完善 E2E：普通用户登录主流程、管理后台关键路径、举报处理闭环
- 增加数据库表与字段中文注释覆盖率
- 增加更多权限边界测试

### 低优先级

- 主站视觉细节与交互动效优化
- 后台模块化路由与布局进一步拆分
