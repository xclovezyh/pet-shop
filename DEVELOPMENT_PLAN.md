# 萌宠集市开发计划

本文档是项目后续开发的计划来源。以后继续开发时，优先参考本文件，不再只依赖聊天记录里的临时计划。

## 执行规则

- 每次继续开发前，先查看本文件和 `git status`。
- 功能点按模块维护；用户指定模块时，优先从指定模块挑选任务。
- 已完成的任务保留在原模块位置，并使用删除线标记，例如 `~~已完成任务~~`。
- 每轮开发优先选择 5-6 个小功能，按难度从低到高推进。
- 一轮小功能完成后再统一验证，至少包含后端构建、前端类型检查、前端构建；涉及接口时补充接口验证。
- 验证通过后更新本文档，把完成项改成删除线，并提交到当前个人分支。
- 如果发现新需求或拆分出的子任务，追加到对应模块，不覆盖历史计划。

## 模块优先级使用方式

以后可以直接指定模块继续开发，例如：

```text
优先做用户模块，按照 DEVELOPMENT_PLAN.md 完成 5-6 个小功能后验证并提交。
```

如果没有指定模块，默认按当前推荐顺序推进：

1. 社区日常与互动模块。
2. 内容安全模块。
3. 搜索筛选模块。
4. 图片与上传模块。
5. 交易流程模块。
6. UI 与体验模块。
7. 管理后台、测试和部署。
